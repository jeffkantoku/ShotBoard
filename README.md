You import a script in pdf or txt format and it breaks it down and schedules the days. You still have to tag a lot of elements like PROPS, etc. manually, but it works for my purposes.

You need Python installed. And I also installed pdfplumber as a pdf parser.

This tool was built for a MacBook Pro laptop. So the GUI might be for MacOS only. I'm not sure. It runs in the Terminal and starts from a command line inferface.


Workflow: import a PDF screenplay, auto-detect scenes, tag breakdown elements (cast, props, wardrobe, etc.), build a colored stripboard, insert day breaks, and export reports including a basic Day-Out-of-Days. It uses Python's built-in tkinter for the GUI and pdfplumber for parsing. First, let's make sure your environment is good. 

Open Terminal and run: python3 -c "import pdfplumber; print('pdfplumber OK', pdfplumber.version)" python3 -c "import tkinter; print('tkinter OK')"  If pdfplumber fails, install it with python3 -m pip install pdfplumber. If tkinter fails (common with Homebrew Python), run brew install python-tk. The python.org installer already includes tkinter. The app still runs without pdfplumber — you just won't be able to import PDFs (you can import plain text scripts instead). 

If you download ShotBoard (scheduler.py and budget.py) to your Desktop then you can run the app by pasting this into the Terminal: 

cd ~/Desktop/ python3 scheduler.py 

How to use it once it's running. Go to File → Import PDF Script and pick a standard-format screenplay; the app scans for scene headings (INT./EXT.) and auto-creates one scene per heading, estimating page lengths in eighths. In the Breakdown tab, click a scene, edit its fields, and tag elements by choosing a category, typing a name, and pressing Enter (cast names you tag here will flow onto the stripboard and into the Cast List and Day-Out-of-Days reports). In the Stripboard / Schedule tab you reorder scenes with the move buttons and insert Day Break rows to divide the shoot into days; strips are colored by the industry convention (white day-interior, yellow day-exterior, blue night-interior, green night-exterior). The Reports menu exports a one-line schedule, breakdown sheets, a cast list, and a basic DOOD as text files. Projects save to a .sbproj JSON file so you can reopen and keep working.   A few honest limitations to set expectations. Scene detection works well on properly formatted text-based PDFs but won't work on scanned/image PDFs (those need OCR, which isn't included). The page-eighths figure is an estimate from line counts, so you'll want to adjust some manually. Element tagging is manual rather than auto-suggested, and the scheduling is manual reordering rather than auto-optimization by cast/location availability.  It also includes automatic cast suggestion from ALL-CAPS character cues, location-based day-break suggestions, and printable PDF/HTML report export. A note on the PDF export. Direct .pdf output uses reportlab, which is optional — install it with python3 -m pip install reportlab. If it isn't installed, the app falls back to exporting a print-optimized HTML file that you open in any browser and save as PDF via Cmd+P → "Save as PDF" (no dependency needed, and it respects landscape orientation for the Day-Out-of-Days). You can also always export plain .txt.  A quick tour of the capabilities. After importing a script you'll be offered automatic cast detection right away; you can also trigger it any time from Tools → "Auto-Tag Cast (all scenes)" or per-scene with the "Suggest Cast (this scene)" button in the Breakdown tab. The detector pulls ALL-CAPS character cues (handling extensions like (V.O.) and (CONT'D) and filtering out transitions, sluglines, and common false positives), then shows you a checklist so you approve names before anything is tagged — un-check things like BANG or RING that are really sound effects. On the Stripboard tab, "Suggest Day Breaks" asks for a target pages-per-day and whether to keep same-location scenes together, then reorders the board and drops in day breaks accordingly; each day header shows its date and total eighths, and you can still fine-tune by hand afterward. Every report in the Reports menu now lets you choose PDF, printable HTML, or text in the save dialog, with the Day-Out-of-Days automatically set to landscape. One refinement worth knowing: the stripboard now renders day headers as selectable rows tied to their breaks, so to delete a break you select its day-header row and click "Remove Break." 

The budget.py module that is integrated into the ShotBoard Scheduler works within the Scheduler app, but can also be run independently. It exports budgets as Excel files. Install the Excel library first: 

pip3 install openpyxl 

Adding budgeting fits naturally alongside a scheduler — film production scheduling and budgeting are tightly linked (strip-board days drive crew/equipment costs). Here's a robust, self-contained budgeting module built with Tkinter (to match your existing scheduler.py) that exports a properly formatted .xlsx via openpyxl. The budgeting module This follows the standard film budget hierarchy: Above-the-Line, Below-the-Line Production, Post-Production, and Other, with editable line items, automatic subtotals, fringe/contingency calculations, and a styled Excel export. What makes it "robust" The structure mirrors how a real production accountant works: a four-tier hierarchy (section → category → line item), with qty × rate math, editable units (Days/Weeks/Hours/Flat), and automatic rollups at every level. The export applies payroll fringes and contingency as percentages on top of direct costs — the two adjustments every real film budget needs — and produces a styled, frozen-header .xlsx that opens cleanly in Excel, Numbers, or Google Sheets. 

How the workflow now flows You import a script, tag cast, and build your days with Suggest Day Breaks as usual. When you open the budget and edit a line item — say "Location fees" or "Camera package rental" — the Link to schedule dropdown will show Total Shoot Days, Night Days, and one Loc: … entry per set in your current board. Pick one, and that item's Qty auto-fills from the live count. Whenever you reshuffle the board or change day breaks, hit Sync from Schedule in the budget window and every linked Qty refreshes. One thing worth knowing: the provider is read fresh each time the budget calls it (both when you open the link dropdown and when you press Sync), so it always reflects the board's current state — there's no stale snapshot. The two are still fully decoupled, so running python3 budget.py on its own continues to work with its built-in demo schedule.
